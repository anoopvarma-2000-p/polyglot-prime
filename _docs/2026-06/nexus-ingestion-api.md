# nexus-ingestion-api

## Overview

`nexus-ingestion-api` is a Spring Boot service purpose-built for multi-protocol healthcare message ingestion. It receives HL7 v2.x messages over MLLP and raw TCP, SOAP/IHE XDS documents over HTTP, and routes them to AWS S3 and SQS for downstream processing. It uses Apache Camel for message routing and Netty for low-level TCP handling.

- **Artifact ID**: `nexus-ingestion-api`
- **Group ID**: `org.techbd`
- **Version**: `0.1167.0`
- **Packaging**: JAR (Spring Boot executable)
- **Java**: 21 LTS
- **Spring Boot**: 3.3.3
- **Apache Camel**: 4.10.0

## Supported Protocols

### 1. MLLP / HL7 v2.x (TCP)

The service listens for HL7 v2.x messages using **MLLP** (Minimal Lower Layer Protocol) over raw TCP. Apache Camel's `camel-mllp` component handles framing and acknowledgement.

- Configured via `MllpHealthIndicator` and `PortConfig`
- Acknowledgements generated by `AcknowledgementService`
- Port assignments are dynamic, driven by `PortConfigApplierService`

### 2. Raw TCP (Netty)

`NettyTcpServer` starts Netty-based TCP listeners. `NettyProxyConfig` and `NettyProxyDecoderConfig` handle HAProxy PROXY protocol v1/v2 for source IP preservation behind load balancers.

- `PortBasedPaths` maps ports to routing contexts
- `ProxyProtocolParserUtil` decodes the PROXY protocol header

### 3. SOAP / IHE XDS (HTTP)

Spring-WS exposes SOAP endpoints for IHE ITI transactions:

| Endpoint Class | IHE Transaction | Description |
|---------------|----------------|-------------|
| `PnrEndpoint` | ITI-41 (Provide and Register) | Receives CCD-A / XDS documents |
| `PixEndpoint` | ITI-8/9 (PIX) | Patient Identity Cross-referencing |
| `XdsRepositoryController` | XDS.b | HTTP-based XDS document submission |

WS-Addressing headers are validated by `WsaHeaderInterceptor`.

MTOM (Message Transmission Optimization Mechanism) is supported for large binary payloads via `MtomSoapResponseStrategy`.

### 4. REST / Data Hold

`DataHoldController` and `DataIngestionController` provide REST endpoints for direct data submission and hold queue management.

## Message Processing Pipeline

All ingested messages (regardless of protocol) flow through a common processing pipeline:

```
Incoming Message
      │
      ▼
MessageProcessorService        ← coordinates steps
      │
      ├─► S3UploadStep          ← store raw payload in AWS S3
      │
      └─► SqsPublishStep        ← notify downstream via AWS SQS FIFO queue
```

`MessageProcessingStep` is the common interface. Steps are applied in order by `MessageProcessorService`.

`MetadataBuilderService` enriches messages with provenance metadata before storage.

## Port-Based Routing

The service supports multiple concurrent TCP listeners, each mapped to a routing profile:

| Class | Role |
|-------|------|
| `PortConfig` | Defines per-port configuration (bucket, queue, tenant, message type) |
| `PortConfigApplierService` | Applies configurations to started ports |
| `PortConfigResolver` | Resolves the correct config for a given port number |
| `PortConfigAttributeResolver` | Resolves specific attributes (bucket, queue, etc.) |
| `BucketResolverImpl` | Resolves the S3 bucket for a port |
| `QueueResolverImpl` | Resolves the SQS queue URL for a port |
| `DataDirResolverImpl` | Resolves the local data directory for a port |
| `RouteParamResolver` | Resolves Camel route parameters |

## Message Grouping

For batch and stream deduplication, messages can be grouped before publishing:

| Strategy Class | Grouping Key |
|---------------|-------------|
| `IpPortMessageGroupStrategy` | Source IP + Port |
| `MllpMessageGroupStrategy` | MLLP framing boundaries |
| `TcpMessageGroupStrategy` | TCP session boundaries |
| `SourceMsgTypeGroupStrategy` | HL7 message type (MSH-9) |
| `TenantMessageGroupStrategy` | Tenant identifier |
| `MessageGroupService` | Selects and applies the appropriate strategy |

## SOAP Response Strategies

The service uses a strategy pattern for SOAP responses:

| Strategy Class | Use Case |
|---------------|---------|
| `DefaultSoapResponseStrategy` | Standard synchronous SOAP response |
| `MtomSoapResponseStrategy` | MTOM-encoded binary responses |
| `TruBridgeMtomSoapResponseStrategy` | TruBridge-specific MTOM responses |
| `SoapResponseStrategyFactory` | Selects strategy based on request characteristics |
